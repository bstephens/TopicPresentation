Generic Subprograms

 

 

  • Creating generic subprograms is one way to increase the reusability of software
  • Useful when the same algorithm can be used for different data types e.g. sorting arrays
  • Polymorphic subprogram takes parameters of different types on different activations
  • Overloaded subprograms don't need to behave similarly and they provide ad hoc polymorphism
  • Parametric polymorphism is provided by a subprogram that takes generic parameters that are used in type expressions that describe the types of parameters of the subprogram

C# generic method example

class MyClass {
   public static T DoIt<T>(T p1) {
      ...
   }
}

int myInt = MyClass.DoIt(17);
string myString = MyClass.DoIt<string>("apples");
