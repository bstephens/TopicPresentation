Parameter-Passing Methods

 

Semantic Models of Parameter Passing

  • In mode - parameters can receive data from the corresponding actual parameter
  • Out mode - parameters can transmit data to the actual parameter
  • Inout mode - parameters can do both 

Implementation Models of Parameter Passing

  • Pass-by-value
    • Value of the actual parameter is used to initialize the corresponding formal parameter
    • Formal parameter acts as a local variable in the subprogram
    • Main advantage is that it is very fast for scalars in both linkage cost and access time
    • Main disadvantage is that the copy operation can be very costly if the parameter is large e.g. an array with many elements
  • Pass-by-result
    • Implementation model for out-mode parameters
    • Similar advantages and disadvantages to pass-by-value
    • Can result in parameter collision
    • Makes programs unportable between an implementation that evaluates addresses for out-mode parameters at the beginning of execution and an implementation that evaluates addresses at the end

Parameter collision example

void Fixer(out int x, out int y) {
   x = 17;
   y = 35;
}

...

f.Fixer(out a, out a);

  • Pass-by-value-result
    • Sometimes called pass-by-copy
    • Value of actual parameter is used to initialize the formal parameter which acts as a local variable to the subprogram
    • Value of formal parameter is copied back to the actual paramter at subprogram termination
    • Shares the same copy disadvantages as pass-by-value and pass-by-result
  • Pass-by-reference
    • Transmits an access path to actual parameter, usually a memory location, to the called subprogram
    • Main advantage is that the passing process is very efficient
    • Disadvantages
      • Access to formal parameters is slower because it is indirect
      • There could be inadvertent changes made to the actual parameter if only one-way communication is necessary
      • Aliases can be created which makes program verification harder and reduces program reliability
  • Pass-by-Name
    • The actual parameter is textually submitted for the correspoding formal parameter in all its occurrences in the subprogram
    • Used at compile time by the macros in assembly languages
    • Not common so it is not discussed further

Implementing Parameter-Passing Methods

  • In most contemporary languages, parameter communication takes place through the run-time stack
  • Pass-by-value parameters have their values copied to stack locations which serve as storage for the formal parameters
  • Pass-by-result parameters are the opposite of pass-by-value
  • Pass-by-value-result parameters can be implemented as a combination of pass-by-value and pass-by-result
  • Pass-by-reference simply places the address in the stack
